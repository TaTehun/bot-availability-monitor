// technical_highlights

DETECTION
Process + TCP Correlation
remoting_host.exe stays alive between sessions. Status is only IN_USE when the process PID also has an established TCP connection via Get-NetTCPConnection.
RESILIENCE
Atomic Write + .tmp Auto-Recovery
Each write goes to .tmp then renames atomically. On OFFLINE or parse failure, widget checks if .tmp is newer and auto-recovers via os.replace — handles SMB file lock gracefully.
UX
Silent Always-on-Top Widget
Launched via widget.vbs → widget.bat → pythonw. No console window, no taskbar entry. pywebview renders the HTML/CSS UI in a frameless 400×280 window pinned on top.

// status_reference

AVAILABLE
No active session
remoting_host.exe not running, or running with no established TCP connection on its PID. Bot is online and idle.
IN_USE
Active remote session
Process running AND has at least one established TCP connection — a Google Remote Desktop session is currently active.
OFFLINE
Unreachable or stale
JSON file missing (3+ consecutive misses), or mtime older than 90 seconds. Bot is off, network is down, or Task Scheduler stopped.

GRD Session Detection monitor_BotN_example.ps1 
# Process alone is not enough —
# remoting_host stays alive between sessions.
# Confirm an active TCP connection on its PID.
$processes = Get-Process -Name "remoting_host" `
    -ErrorAction SilentlyContinue

if ($processes) {
    $pids = $processes | Select-Object -ExpandProperty Id
    $conn = Get-NetTCPConnection `
        -State Established `
        -ErrorAction SilentlyContinue |
        Where-Object { $pids -contains $_.OwningProcess }
    if ($conn) { $status = "IN_USE" }
}

Concurrent Read via ThreadPoolExecutor widget_example.py · Api.get_status() 
# Read all 6 bot files in parallel —
# network share I/O would serialize otherwise.
def get_status(self):
    with ThreadPoolExecutor(max_workers=6) as executor:
        results = list(
            executor.map(self._read_bot, BOTS)
        )
    return results
FastAPI Status Endpoint app_example.py · read_bot() 
# Web server reads JSON timestamp from content,
# not mtime — stateless per-request, no cache.
def read_bot(bot):
    file = STATUS_DIR / f"{bot}.json"
    if not file.exists():
        return {"bot": bot, "status": "OFFLINE", ...}
    try:
        data = json.loads(file.read_text(encoding="utf-8-sig"))
        updated = datetime.strptime(
            data["updated"], "%Y-%m-%d %H:%M:%S"
        )
        if (datetime.now() - updated).total_seconds() > 30:
            data["status"] = "OFFLINE"
        data.setdefault("last_connected", None)
        return data
    except:
        return {"bot": bot, "status": "ERROR", ...}

@app.get("/api/status")
def status():
    return [read_bot(b) for b in BOTS]

Detection Precision: Process vs TCP
TJ
Just checking remoting_host is not enough — the process stays alive between sessions. Need to confirm an actual TCP connection on that PID.
C
Added Get-NetTCPConnection -State Established filtered by OwningProcess. Status is only IN_USE when both conditions are true simultaneously.
mtime vs JSON Timestamp for Stale Check
TJ
Should the 30s stale check use the updated field inside the JSON, or the file's mtime?
C
mtime is safer — it reflects when the file was actually written, not a value the bot script controls. A bug in the script that writes a wrong timestamp won't fool the stale guard.
Silent Launch via VBS
TJ
widget.bat flashes a console window on launch. Need to start it with no visible terminal and no taskbar entry.
C
Added widget.vbs: WScript.Shell.Run with window style 0 hides the window entirely. Double-clicking VBS starts the widget silently.
OFFLINE Auto-Recovery via .tmp
TJ
When .json is stale (OFFLINE) or fails to parse, and .tmp is newer — replace .json with .tmp automatically instead of waiting.
C
Used os.replace instead of unlink + rename — SMB held a file lock (WinError 32) on the old .json, but os.replace overwrites atomically without needing to delete first.
